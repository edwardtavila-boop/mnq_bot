## Invariants the simulator MUST preserve

1. Every state transition is journaled before it is reflected in memory.
2. Replay of the journal reproduces the in-memory world exactly
   (`OrderBook.from_journal`, `net_positions_from_journal`).
3. The Firm skill markdown is consumed — never imported as Python — so that
   the in-progress Firm codebase can swap in without any refactor here.
4. No variant is considered "shippable" unless its bootstrap CI excludes zero
   **and** its n ≥ 8 **and** a falsification criterion was written ahead of
   the run (`reports/firm_reviews/<variant>.md`).
5. Turnover drift z-score must stay inside ±3σ of the per-variant expected
   distribution — otherwise a `DRIFT_ALERT` event fires.

## Where the edge search happens

* `scripts/walk_forward.py` — rolling-window variant search across the real
  MNQ 1m data (found in `data/real_mnq_1m_rth`). Sweeps `StrategyConfig` with
  bootstrap CI and prints only variants whose expectancy CI excludes zero.
* `scripts/firm_vs_baseline.py` — Firm-filtered signals vs a naive momentum
  baseline on the same tape. If the Firm filter doesn't lift expectancy, it
  has no right to slow us down.
* `scripts/calibration.py` — Brier / log-loss on `ml_scorer` probabilities vs
  journal outcomes. A poorly-calibrated scorer is a de facto false signal.
* `scripts/bayesian_expectancy.py` — per-variant / per-regime Beta posteriors
  updated from FILL_REALIZED, used to re-rank variants as evidence accrues.
