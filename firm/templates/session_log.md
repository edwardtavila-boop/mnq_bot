# Daily Session Log

Minimum data capture. Track trades taken AND trades skipped — both are data.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DAILY SESSION LOG — [DATE]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PRE-SESSION
Regime: _______
Scheduled catalysts: _______
Account state: _____% of high-water
Daily risk budget: ____R
Mental state (1-10): _______
  (If ≤5, stand down.)

TRADES TAKEN
#  Time  Setup  Entry  Stop  Tgt  Exit  R   Notes
1  ____  ____  ____   ____  ___  ____ __  ____
2  ...

TRADES SKIPPED (counterfactual — critical for learning)
#  Time  Setup  Why skipped             Would have been: +/- ?R
1  ____  ____  ___________________     _______
2  ...

DIVERGENCE FROM PLAN
Followed system? (Y/N)
Any overrides? List: _______
Rule violations? _______

POST-SESSION
Total P&L: ____R
Peak exposure: ____R
Worst trade: ____R  Best trade: ____R
vs. daily budget: _______

ONE LESSON FROM TODAY (write BEFORE looking at P&L):
_________________________________________
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Most traders only log executed trades and therefore only learn from half the data. The counterfactual log is the half they're missing — and it's often the more important half.
