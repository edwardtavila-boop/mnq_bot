# Override Rationale

Every time the PM overrides an agent's veto or recommendation, this is completed BEFORE the override takes effect. Silent overrides are process violations.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
OVERRIDE RATIONALE
Date/Time: _______   Strategy: _______
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

AGENT BEING OVERRIDDEN: _______

THEIR POSITION (verbatim):
_________________________________________
_________________________________________

MY OVERRIDE:
_________________________________________

THE THREE QUESTIONS (must answer all):

1. What specific information do I have that this agent does not?
   _________________________________________

2. What is the empirical basis (not "my gut")?
   _________________________________________

3. What outcome would cause me to REVERSE this override?
   _________________________________________

This override is logged. Post-mortem evaluates at 30 days.

Signed: _______
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

If you cannot answer question 3 with a specific numeric or event trigger, you do not have an override — you have wishful thinking. Go with the agent.
